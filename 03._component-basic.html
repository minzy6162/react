컴포넌트의 종류

  • 클래스 기반 컴포넌트

    • ES6에서 제공하는 클래스 문법 이용하여 컴포넌트 정의 가능
xxxxxxxxxx
// 다음 모듈 불러오기 구문은 매 예제마다 삽입
import React, {Component} from 'react'
import ReactDOM from 'react-dom'
​
// 클래스 기반 컴포넌트 정의
class MyComponent extends Component {
    // 렌더 함수 정의
    render() {
        // JSX를 이용하여 컴포넌트에서 보여줄 내용을 정의
        return (
            <div>My Component</div>
        )
    }
}

컴포넌트에서 가장 중요한 함수인 render 함수(=메소드)의 역할

  • render 함수는 반드시 정의해야 함

  • render 함수를 통해 가상 DOM에 변경 사항이 적용되고 이후 실제 DOM에 적용됨

  • 일반적인 태그 혹은 다른 컴포넌트를 JSX를 이용하여 구성 가능

  • 반드시 하나의 루트 태그 내부에 모든 태그가 존재해야 함

    • DOM 노드의 효율적인 관리 및 비교를 위해서 존재하는 제약

    • 일반적으로 div를 써서 내부 태그(및 서브 컴포넌트)를 묶어주지만 만약 쓰고 싶지 않은 경우 React.Fragment 사용 가능

      • https://reactjs.org/docs/fragments.html

      • 혹은 React.Fragment 대신에 <></> 사용 가능 (short syntax)

        • 반드시 사용해야 하는 case가 존재함

          • 가령, td 태그 위에는 tr이 존재해야 하므로 부모 태그가 존재해서는 안 됨
        • xxxxxxxxxx
          class Columns extends React.Component {
            render() {
              return (
                <React.Fragment>
                  <td>Hello</td>
                  <td>World</td>
                </React.Fragment>
              );
            }
          }

 

컴포넌트 속성 (Props)

 

props

xxxxxxxxxx
props-basic-1.js

  • 컴포넌트 속성 => 컴포넌트에 전달할 인자값 (함수에 전달하는 인자값과 비슷한 역할을 수행)

    • (일반적으로) 전달한 props 값은 컴포넌트를 구성하는데 사용되는 정보

  • 전달된 props 값은 수정이 불가능 (읽기 전용 값)

    • 새로운 값을 대입하는 것이 불가능하다는 의미이며 전달한 props 값이 객체인 경우 엄밀히 말하면 내용 수정 가능 (단, 가능하다고 해도 절대로 props 값을 수정하지 않아야 함!)

      • xxxxxxxxxx
        const user = {}
        // 가능
        user.name = "John"
        // 불가능 (Uncaught TypeError: Assignment to constant variable.)
        user = {}

    • 컴포넌트는 props 값에 수동적으로 대응하여 내용을 표시할 뿐, 절대로 값을 바꿔서는 안 됨 (props 값에 대해서는 순수 함수처럼 side effect가 없도록 작동해야 함)

      • All React components must act like pure functions with respect to their props.

    • 자바스크립트에서는 함수를 값으로 취급하므로 속성값으로 함수도 전달할 수 있음 (이후 특정 컴포넌트의 state를 바꾸는 메소드를 정의하여 해당 메소드를 자식 컴포넌트로 전달하여 부모 컴포넌트의 state 변경이 이루어지도록 함)

  • 만약에 prop 값을 설정하되, 값 대입과 관련된 내용이 없다면 true가 전달됨

xxxxxxxxxx
{/* autocomplete 값은 true */}
<MyTextBox autocomplete />
​
{/* 위의 코드와 같은 역할 수행 */}
<MyTextBox autocomplete={true} />
  • 전개 연산자(...) 사용하여 props 값을 전달
xxxxxxxxxx
// 보통의 props 값 전달 방식
<ComponentWithMultipleProps value1={1} value2='React' whatever={1.234} />
​
const propsObj = {
    value1: 1,
    value2: 'React',
    whatever: 1.234
}
// 객체에 전개 연산자 사용하여 props 값 전달 (위의 컴포넌트 사용 코드와 동일)
<ComponentWithMultipleProps {...propsObj} />

함수 컴포넌트

xxxxxxxxxx
function-component.js
  • 구버전 리액트에서는 단순히 props 값을 전달받아 렌더링만 하는 용도로 함수 컴포넌트가 사용됨
  • 리액트 16.8 버전 이후 리액트 훅(Hook)이 도입되어 함수 컴포넌트를 통해서도 클래스 컴포넌트에서 지원하는 기능(상태 관리, 라이프 사이클 메소드)을 사용할 수 있도록 수정함

const Greeting = (props) => {
    return (<h1>Hello, {props.name}</h1>)
}
