컴포넌트 상태 (State)

  • 변경이 불가능한 props와는 다르게 컴포넌트에서 관리할, 변경이 가능한 state 객체를 정의 가능

    • props가 부모 컴포넌트로부터 전달받는 수동적인 데이터(정보)에 가깝다면, state 객체는 주체적, 능동적으로 컴포넌트 스스로가 관리할 데이터에 가까움

                    })
                }}>{this.state.value1}</button>

  • 생성자 메소드(constructor)에 state 객체 정의 가능

    • 해당 객체는 이후 불변값으로 취급

      • 객체를 불변값으로 취급하라는 의미는 객체의 내용의 변경이 필요할 때 해당 객체를 직접 수정하지 말고, 새로운 객체로 교체하면서 수정하여 이전에 참조하던 객체의 내용이 변경되지 않도록 하라는 의미 (=이전 객체의 불변성 유지)

    • 절대로 상태값을 직접 변경하면 안 됨

      • 컴포넌트에 정의된 setState 함수를 이용하여 상태 변경
      • setState를 호출해서 상태를 변경해야 내부적으로 상태 병합 작업이 이루어지고 render 메소드를 재호출함

setState 메소드

  • 동기적으로 동작하지 않고 비동기적(asynchronous)으로 동작함을 유의!

    • 즉, setState는 순차적으로 동작하는 메소드가 아님을 유의
    • 큐에 변경될 사항들을 저장하고 처리하므로, 내부적으로 상태가 실제로 바뀌는 시점이 조금 늦어질수도 있음을 유의

  • 내부적으로 객체 상태를 병합(merge)하는 작업을 수행하므로 객체의 내용을 구성할 일부분 정보만 전달해도 무방함

xxxxxxxxxx
this.state = {
    value1: "hello",
    value2: 1000,
    inner: {
        value3: null
    }
}

  • 위의 코드에서 setState 메소드를 이용하여 value1 속성값을 변경하고 싶은 경우 value1를 속성으로 포함하고 있는 객체만 전달하면 됨

    • 전달한 객체와 기존 객체를 병합(merge)하여 새로운 객체를 생성하는 방식으로 동작
    • https://stackoverflow.com/questions/42731453/deep-and-shallow-merge-in-javascript
xxxxxxxxxx
// Q) value1을 "Bye"로 변경하되, 나머지 속성값은 그대로 두려면?
// 다음과 같이 value1 속성을 포함한 객체만 전달하면 내부적으로 병합 작업을 수행
this.setState({
    value1: "Bye"
})

  • 만약 이전 상태값에 기반하여 값을 변경(ex: 카운터)해야 할 경우에는 콜백 함수를 전달해서 이전 상태값을 받아와 해당 상태값을 이용하여 값을 변경해야 함!

    • 리액트 내부적으로 퍼포먼스 향상을 위해서 여러번 setState를 호출해도 하나의 업데이트 과정에서 모두 처리하도록 최적화할 수 있음을 유의! (setState가 바로 적용되는게 아니고 큐에 쌓여서 처리하기 때문)

  • 가령 value2의 값을 이전 값에서 1을 뺀 값으로 변경하고 싶은 경우 다음과 같이 setState 메소드에 콜백 함수를 전달하는 방식으로 처리해야 함

xxxxxxxxxx
// setState 메소드로 콜백 함수를 전달
this.setState(state => {
    return { value2: state.value2 - 1 }
})

import React, { Component } from 'react'
import ReactDOM from 'react-dom'
​
class FunctionalSetStateDemo extends Component {
    constructor(props) {
        super(props)
​
        this.state = {
            count: 0
        }
    }
    brokenInc = () => {
        // 반복문을 이용하므로 10이 증가되어야 하지만, 실제로 확인해보면 1만 증가
        for(let i=0;i<10;i++) {
            this.setState({ count: this.state.count + 1 })
        }
    }
    brokenLog = () => {
        console.log(this.state.count)
        // setState는 순차적으로 실행되지 않고 비동기적으로 작동함
        // https://stackoverflow.com/questions/36085726/why-is-setstate-in-reactjs-async-instead-of-sync
        this.setState({ count: this.state.count + 1 })
        // 따라서 바로 변화가 반영되지 않을 수 있음
        console.log(this.state.count)
    }
    normalInc = () => {
        for(let i=0;i<10;i++) {
            // 만약 이전 상태에 기반하여 업데이트를 할 경우에는 반드시 콜백 함수를 전달하는 형태로 setState 호출
            // 해당 함수로 이전 상태값이 전달됨
            this.setState((prevState) => ({ count: prevState.count + 1 }))
        }
    }
    normalLog = () => {
        // Q) 정상적으로 작동하도록 brokenLog 코드 참조하여 재작성하기
    }
    render() {
        return (
            <div>
                <p>{this.state.count}</p>
                <button onClick={this.brokenLog}>brokenLog</button>
                <button onClick={this.brokenInc}>brokenInc</button>
                <button onClick={this.normalInc}>normalInc</button>
            </div>
        );
    }
}

ReactDOM.render(<FunctionalSetStateDemo difficulty={1} />, document.getElementById("root"))

import React, { Component } from 'react'
import ReactDOM from 'react-dom'
​
class ShallowMergeDemo extends Component {
    constructor(props) {
        super(props)
​
        this.state = {
            name: "John",
            age: 20,
            address: {
                city: "New York",
                street: "20 W 34th St"
            },
            /*
            inner: {
                data1: 1000,
                data2: 2000,
                nestedInner: {
                    data3: 2000,
                    data4: 3000,
                    text: "Hello"
                }
            }
            */
        }
    }
​
    brokenUpdate = () => {
        // 중첩 객체의 내용을 변경할 경우, setState 메소드는 얕은 병합(shallow merge)을 수행하므로 한 단계 더 들어간 중첩 객체 내용은 대체(replace)되어 버리는 현상이 발생함
        this.setState({
            address: { street: "new street info" }
        })
    }
​
    normalUpdate = () => {
        // 이전 상태를 전달받도록 콜백 함수 전달 후,
        this.setState( state => {
            // 중첩 객체의 상태를 전개 문법(spread syntax)를 이용해서 복사한 후, 새 값을 대입하는 식으로 수동 merge 작업 수행이 필요
            return {
                address: {
                    ...state.address,
                    street: "new street info"
                }
            }
        })
    }
​
    normalUpdateInner = () => {
        // Q) nestedInner 객체의 text를 "World"로 변경하되, inner 내부의 데이터를 모두 온전히 보존할 수 있도록 수정하게 코드 작성
    }
​
    render() {
        return (
            <div>
                <p>{JSON.stringify(this.state)}</p>
                <button onClick={this.brokenUpdate}>broken update</button><br />
                <button onClick={this.normalUpdate}>normal update</button><br />
                <button onClick={this.normalUpdateInner}>normal update inner</button>
            </div>
        );
    }
}
​
ReactDOM.render(<ShallowMergeDemo />, document.getElementById("root"))

  • 내부적으로 리스트에 포함된 특정 요소를 유일하게 구분할 문자열 키를 이용하여 퍼포먼스를 최적화하는데 사용

    • 목록에 포함된 항목 중 어떤 항목이 추가, 변경, 삭제되었는지 여부를 빨리 파악하고 반영하기 위해서 내부적으로 키를 활용함

  • 보통 데이터베이스에서 가져온 데이터를 보여주게 되므로 가장 좋은 키는 데이터베이스에 저장된 각 행의 primary key

    • 단, 데이터베이스 접근이 힘든 예제 코드에서는 임시로 uuid를 할당하여 키값으로 활용할 예정

  • 배열 항목의 위치(=인덱스)를 이용하는 방법은 권장되지 않음

    • 왜냐하면, 기본적으로 키가 변하지 않는다(stable)는 전제 조건하에 사용하는 것인데, 위치는 변경될 수도 있으므로 좋은 키가 아님

      • 따라서, 목록의 항목 위치가 변경되지 않는다고 가정할 수 있는 경우에는 사용해도 되긴 하지만(?) 그래도 PK를 쓰는게 좋음

  • 키를 제공하지 않을 경우 경고 메시지가 콘솔에 출력됨

    • 또한, 키 속성을 부여하지 않을 경우, 내부적으로는 그냥 항목의 위치(=인덱스)를 키로 사용함

  • 키는 특정한 한 개의 목록 내부에서만 유일하면 됨 (즉, 키 값이 전역적으로 유일해야 할 필요는 없음)
